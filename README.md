# MemoMind Assistant

MemoMind Assistant 是一个面向个人与企业场景的多模态智能助手，帮助用户进行对话、知识检索、记忆管理和工具调用。

项目以“让助手记得住、找得到、做得到”为目标，将大语言模型、RAG、长期记忆和可控工具执行结合起来，提供可扩展的智能体基础能力。

## 项目亮点

- **多模式智能体**：支持普通对话、知识库问答、单工具调用和多工具协作，并根据用户问题自动选择合适的处理方式。
- **混合检索增强**：结合向量检索、关键词检索和知识图谱，提升复杂问题的召回质量，并在部分基础设施不可用时自动降级。
- **三层记忆体系**：通过短期记忆、长期记忆和用户偏好记忆，让助手能够理解当前上下文并保留跨会话信息。
- **记忆质量治理**：支持记忆去重、合并、重要性衰减和过期淘汰，减少重复或低价值信息对回答的干扰。
- **工具调用与任务恢复**：支持搜索、RAG、命令执行等工具，并提供任务快照、重试和可恢复执行能力。
- **安全沙箱执行**：支持 Docker、Local 和 Mock 执行后端，可配置命令校验与资源限制，方便在不同环境中使用和测试。
- **工程化设计**：核心基础设施采用可选依赖和优雅降级策略，支持从轻量本地模式逐步扩展到完整服务部署。

## 快速开始

### 本地运行

```bash
go mod tidy
go run ./cmd/server
```

启动后访问：`http://localhost:8090`

### 使用 Docker

```bash
docker compose up -d --build
```

### 配置

复制或编辑 `config/config.yaml`，按需配置以下服务：

- LLM API：用于对话、规划和偏好提取
- Embedding API：用于语义检索和记忆匹配
- Tavily API：用于联网搜索，可选
- PostgreSQL、Milvus、Elasticsearch、Neo4j、Kafka：按功能需要启用，可选

基础设施连接失败时，系统会根据能力自动降级到可用模式。

## 技术栈

- Go

- PostgreSQL、Milvus、Elasticsearch、Neo4j、Kafka

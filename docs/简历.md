\datedsubsection{\textbf{MemoMind Assistant 智能记忆助手}}{}{}
\role{核心开发}{Go, LLM Agent, RAG, Memory, Docker, PostgreSQL}
\begin{itemize}
	\item \textbf{智能体编排与流式交互}：构建统一 Agent 调度层，按问题自动路由至对话、知识库、单工具或 ReAct 多工具模式；统一流式与非流式执行链路，通过 SSE 输出过程事件，并支持任务取消、快照与恢复。


	\item \textbf{多租户与上下文治理}：在领域层强制按用户 ID 隔离记忆读写，避免调用方遗漏过滤导致的数据泄露；以 Schema 驱动方式按任务模式装配画像、任务状态、工具状态、安全约束和召回结果，并通过槽位预算控制 Prompt 上下文规模。

	\item \textbf{Hybrid GraphRAG 检索链路}：实现 Markdown/PDF 文档入库、父子分块与代码块保护，融合 Milvus 向量检索、Elasticsearch BM25 和 Neo4j 知识图谱三路召回；使用 RRF 融合、多查询改写、可选 LLM 精排与父块回填，提升复杂技术问题的检索覆盖与回答上下文质量。

	\item \textbf{图式 ReAct 与工具生态}：将多步骤任务建模为 DAG，基于拓扑排序和入度控制并行执行无依赖节点，并支持多检索源竞速、超时、重试、降级与软失败恢复；通过统一注册表接入搜索、命令、文档和 MCP 工具，为后续扩展领域工具及子智能体保留接口。

	\item \textbf{安全执行与工程化部署}：设计 Docker / Local / Mock 可切换沙箱，结合网络隔离、只读根文件系统、能力剥夺、资源限额和命令风险分级，约束工具执行边界。
\end{itemize}
